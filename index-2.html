03. Vue.js 프레임워크 3가지 특징

  1. 컴포넌트 기반의 개발 방식 : 재사용성 / 구현속도 / 코드가독성
  2. MVVN 패턴 : 화면 UI코드와 백엔드 데이터 처리코드를 분리 (View - ViewModel - Model)
  3. 리액트와 앵귤러의 장점을 흡수
  • 데이터바인딩과 화면단위를 컴포넌트 형태로 제공하며, 관련API를 지원하는데 목적
  • 앵귤러에서 지원하는 양방향 데이터 바인딩을 동일하게 제공
  • 컴포넌트 간 통신의 기본은 리액트(단방향데이터흐름) props사용
  • 다른 프런트 엔드 와 비교했을 때 상대적으로 가볍고 빠름
* MVVM 패턴 : MVC패턴에서 기인, 화면 앞단(view)와 뒷단의 DB 처리 및 로직 분리하고 뒷단에서 넘어온 데이터를 Model에 담아 View로 넘겨주는 것

04. Vue.js 코드 맛보기

Vue instance - 뷰 화면을 위해 기본적으로 생성해야 하는 단위

인스턴트 속성, API들

  • el: 인스턴스가 그려지는 화면의 시작점 (특정 HTML태그)
  • template: 화면에 표시할 요소 (HTML, CSS)
  • data:뷰의 반응성이 반영된 데이터 속성
  • methods: 화면의 동작과 이벤트 로직을 제어하는 방법
  • created: 뷰의 라이프사이클과 관련된 속성
  • watch: data에서 정의한 속성이 변화했을때 추가 동작을 수행할 수 있게 정의
